# AI WORK PROTOCOL

Version: 1.0

Status: AUTHORITATIVE

Owner: Founder

---

# Purpose

This document defines the mandatory working methodology for any AI system contributing to EIOS.

The objective is enterprise-quality engineering rather than rapid code generation.

---

# General Principle

Think before implementing.

Analyze before deciding.

Plan before executing.

Validate before continuing.

---

# Standard Workflow

Understand

↓

Analyze

↓

Plan

↓

Recommend

↓

Wait if approval is required

↓

Implement

↓

Validate

↓

Review

↓

Continue

---

# Before Every Task

Determine:

- Objective

- Scope

- Dependencies

- Risks

- Expected Deliverables

- Definition of Done

Do not begin implementation before understanding the task.

---

# Work Package Rule

Break every large objective into:

- Epic

- Sprint

- Step

- Task

- Deliverable

Never attempt uncontrolled large implementations.

---

# Implementation Rule

Implement one logical work package at a time.

Finish it before starting another.

Avoid partially completed functionality.

---

# Review Rule

After implementation verify:

- architecture compliance

- governance compliance

- consistency

- maintainability

- documentation

- quality

---

# Progress Reporting

Whenever appropriate report:

- Current Epic

- Current Sprint

- Current Step

- Completed Work

- Remaining Work

- Risks

- Recommendation

---

# Continuous Improvement

Continuously evaluate:

Can this be:

- simpler?

- clearer?

- safer?

- more maintainable?

- more scalable?

If yes,

recommend improvements.

---

# Stop Protocol

Stop immediately if:

- requirements conflict

- architecture conflicts

- governance conflicts

- assumptions are unsupported

Explain the issue.

Recommend alternatives.

Wait for Founder decision if necessary.

---

# Golden Rule

Do not optimize for producing more code.

Optimize for producing a better enterprise system.
