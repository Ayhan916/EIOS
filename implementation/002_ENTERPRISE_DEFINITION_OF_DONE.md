ID: DOD-0001

Version: 1.0

Status: APPROVED

---

# Purpose

This document defines when an implementation is considered complete within EIOS.

Code completion is not sufficient.

Enterprise readiness is required.

# General Principles

Every implementation shall be:

- correct

- tested

- documented

- observable

- secure

- maintainable

- explainable where applicable

---

# Entry Criteria

Development may begin only if:

- requirements exist

- architecture exists

- dependencies are identified

- acceptance criteria exist

---

# Completion Criteria

A task is complete only if:

- implementation exists

- code review completed

- tests pass

- documentation updated

- benchmarks pass

- security requirements satisfied

- governance requirements satisfied

---

# Required Tests

Every implementation shall include:

- unit tests

- integration tests

- regression tests

Where applicable:

- contract tests

- performance tests

- security tests

---

# Documentation Requirements

Documentation shall include:

- purpose

- dependencies

- interfaces

- limitations

- future extension points

---

# Security Requirements

Implementation shall:

- validate inputs

- enforce authorization

- generate audit events

- generate trace identifiers

- avoid secret leakage

---

# Observability Requirements

Implementation shall expose:

- logs

- metrics

- events

- trace IDs

---

# AI Requirements

AI functionality shall provide:

- evidence references

- confidence score

- reasoning summary

- model version where applicable

---

# Change Requirements

Every completed implementation shall reference:

- Requirement ID

- Architecture ID

- Change ID (if applicable)

---

# Failure Criteria

A task is NOT complete if:

- tests fail

- documentation is missing

- architecture is violated

- governance is violated

- security is incomplete

---

# Golden Rule

Done means enterprise-ready.

Not merely code-complete.
