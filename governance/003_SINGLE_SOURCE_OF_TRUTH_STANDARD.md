# SINGLE SOURCE OF TRUTH STANDARD

ID: GSSOT-0001

Version: 1.0

Status: APPROVED

Owner: Founder

---

# Purpose

This document defines the Single Source of Truth principle for EIOS.

Every enterprise concept shall have exactly one canonical definition.

All other documents shall reference that definition.

---

# Principles

The repository shall avoid:

- duplicated definitions

- duplicated rules

- duplicated mission statements

- duplicated governance principles

- duplicated architecture concepts

---

# Canonical Rule

Every concept shall have:

- one owner

- one canonical document

- one canonical identifier

---

# Reference Rule

Documents shall reference canonical concepts.

Documents shall not copy canonical concepts.

---

# Example

Correct:

Mission

↓

001_FOUNDER_CONSTITUTION

↓

Referenced everywhere

Mission

# Modification Rule

Canonical documents may change only through governance.

Referenced documents should not redefine them.

---

# Conflict Resolution

If two documents conflict:

1. Founder Constitution

↓

2. Enterprise Blueprint

↓

3. Governance Documents

↓

4. Architecture Documents

↓

5. Product Documents

↓

6. Implementation Documents

The higher authority prevails.

---

# Repository Rule

Copying enterprise definitions is prohibited.

Referencing enterprise definitions is mandatory.

---

# Golden Rule

One concept.

One definition.
