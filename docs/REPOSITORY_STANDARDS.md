# EIOS Repository Standards

Version: 1.0

Status: Draft

Owner: Founder

---

# Purpose

This document defines how the repository shall be structured.

Every future document must follow these standards.

---

# Rule 1

One document = one purpose.

---

# Rule 2

One folder = one architectural responsibility.

---

# Rule 3

No duplicate information.

Every concept has exactly one source of truth.

---

# Rule 4

Every document shall contain:

* Title
* Version
* Status
* Owner
* Purpose
* Dependencies
* Related Documents

---

# Rule 5

Documents shall reference each other instead of duplicating content.

---

# Rule 6

Every architecture decision shall be stored as an ADR.

---

# Rule 7

Every strategic decision shall be stored as an FDR.

---

# Rule 8

Every future implementation must reference:

* Specification
* Architecture
* Decision Record

---

# Rule 9

Code never defines architecture.

Architecture defines code.

---

# Rule 10

The repository itself is part of the product and shall be treated as a strategic asset.
