# EIOS Implementation Protocol

Version: 1.0

Status: Draft

Owner: Founder

---

# Purpose

This protocol defines how every implementation inside EIOS must be executed.

Implementation is the final step of a decision process, not the first.

---

# Development Lifecycle

Every request follows this sequence:

Idea

↓

Problem Definition

↓

Business Value

↓

Architecture Analysis

↓

Alternative Solutions

↓

Risk Assessment

↓

Recommendation

↓

Founder Approval

↓

Implementation

↓

Testing

↓

Benchmark

↓

Documentation

↓

Deployment

---

# Mandatory Analysis

Before implementation Claude must answer:

What problem is solved?

Why does this problem exist?

Is implementation necessary?

Can the architecture be simplified?

Are there better alternatives?

---

# Mandatory Deliverables

Every implementation must generate:

* Architecture Impact

* Expected KPI Improvement

* Expected Risks

* Technical Debt

* Rollback Strategy

* Future Improvements

---

# Forbidden Behavior

Claude shall never:

* Implement without analysis

* Change architecture silently

* Introduce hidden dependencies

* Claim production readiness without evaluation

---

# Completion Criteria

A task is only complete when:

* Code exists

* Tests exist

* Documentation exists

* Benchmark exists

* Rollback exists

* Evaluation exists
